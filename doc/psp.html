2. Externe Scripte

- Smarty ==> /core/smarty/Smarty.class.php
- PEAR::DB ==> /core/pear/db.class.php + /core/pear/DB/*.php
- Swift Mailer ==> /core/switftmailer/

3. Module

- Ordner: /modules
- GET Variablen Übergabe an index.php ==> z.B. index.php?mod=admin&do=enter
- Weiterleitung zu Modul in index.php ( per require )

4. User-/Gruppen

- Gruppenrichtlinien via DB, welche userspezifisch in einem Objekt-Array gehalten werden ==> z.B. $user->group['can_access_acp'], $user->id, $user->name, ... (ungüngstig?)

5. Datenbank Struktur

- $config->db_prefix.'_content'
- Tables: Tabellenname_Feldname ==> e.g. user_name,user_id,user_nick,user_icq

6. Codekonventionen

- PEAR Coding Standards (unter Umständen mit leichten Veränderungen...) ==> http://pear.php.net/manual/de/standards.php
- private Variablen/Funktionen mit Unterstrich ==> $_privatevar, function _privatefunc()
- docBlock Kommentierungen (Nicht ganz so heftig, aber mindestens mit @desc) ==> http://pear.php.net/manual/de/standards.comments.php
- 4 Space Tabstops (widerspricht den PEAR Standards!)
- Silbentrennung durch Unterstrich ==> $gruenes_haus, $input_vars, $tpl_handler
- Leerstelle bei PHP Funktionen ==> require ('config.class.php')
- keine Leerstelle bei normalen Funktionen ==> clear_input($string='')
- Editoren: PHPed (Features: phpDocumentor, TurtoiseSVN, FTP, SQL), Weaverslave (Features: ???)
- Monospaced Fonts (z.B. Courier New)

7. Voraussetzungen

- PHP 5.1 und höher
- Datenbank: DBase,FBSQL,iBase,ifx,msql,mssql,mysql,mysqli,oci8,odbc,pgsql,sqllite,sybase,storage
- ca. 100 MB Webspace
